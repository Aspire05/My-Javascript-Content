JAVASCRIPT VARIABLE NAMES

  • DEFINITION
  • SYNTAX
  • EXAMPLE


In JavaScript, variable names (identifiers) must follow certain syntax rules. Here are the key points to consider:


1. Valid Characters:
Variable names can include letters (a-z, A-Z).
They can include digits (0-9), but cannot start with a digit.
Underscore (_) and dollar sign ($) are also allowed.
Other special characters are not allowed in variable names.


2. JavaScript is case-sensitive. This means that myVar and myvar would be treated as two different variables.

3. Reserved Words: Avoid using JavaScript reserved words as variable names. Reserved words have special meanings in the language and cannot be used for other purposes.

4. Camel Case (Conventional): It's a common convention in JavaScript to use camel case for variable names. Camel case means starting with a lowercase letter and capitalizing the first letter of each subsequent concatenated word.
